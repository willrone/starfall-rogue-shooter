## 1. 一句话卖点 (Hook)

**卡通科幻肉鸽射击 × 自动开火 × 无尽撤离**  
操控幸存者在异星战场自动射击潮涌而来的怪物，拾取武器与装备不断变强，撑过无尽波次争取最高撤离评级。

---

## 2. 核心循环 (Core Loop)

```
┌─────────────────────────────────────────────┐
│           进入战斗 (Begin Battle)            │
│  ┌───────────────────────────────────────┐  │
│  │  自动射击 → 杀怪 → 掉落 XP/合金       │  │
│  │  ↓                                     │  │
│  │  升级 → 选择被动强化 (3选1)            │  │
│  │  ↓                                     │  │
│  │  波次结束 → 商店购买武器/装备          │  │
│  │  ↓                                     │  │
│  │  更强 → 更难 → 循环                   │  │
│  └───────────────────────────────────────┘  │
│  ↓ 死亡或主动撤离                          │
│  结算 → 永久进度 (合金/仓库) → 机库整备    │
│  ↓                                         │
│  再次出击 → 更多武器/装备解锁              │
└─────────────────────────────────────────────┘
```

**单局时长**: 8-15 分钟（正常玩家 Wave 6-8 死亡）  
**重玩驱动**: 解锁新武器变体、收集传奇装备、冲击更高波次

## 5. 主要系统设计

### 5.1 战斗系统

**自动射击**: 玩家只需控制移动，武器自动朝最近敌人开火。

**伤害公式** (来自代码):
```
子弹伤害 = 武器基础伤害 × (1 + 角色攻击力加成)
开火间隔 = 1 / (武器射速 × 等级 + 角色攻速 × 0.45)
穿透     = 武器穿透 × 等级 + 角色穿透 × 0.3
弹丸数   = 1 + floor(散射值 / 2.2) 额外弹道
```

**10 种武器**:

| 武器 | 伤害 | 射速 | 穿透 | 散射 | 定位 |
|---|---|---|---|---|---|
| 风暴步枪 | 4.2 | 1.05 | 0 | 0 | 均衡入门 |
| 裂变枪管 | 2.6 | 0.4 | 0.7 | 1.2 | 散射穿透 |
| 轨道无人机 | 1.7 | 0.18 | 0 | 0 | 自动电击 |
| 磁轨炮 | 6.5 | -0.05 | 1.1 | 0 | 高伤穿透 |
| 新星霰弹 | 3.1 | 0.22 | 0 | 1.55 | 近距爆发 |
| 离子长枪 | 5.1 | 0.28 | 0.9 | 0.15 | 穿刺线性 |
| 余烬冲锋枪 | 2.4 | 1.45 | 0 | 0.25 | 高速弹幕 |
| 霜束发射器 | 3.7 | 0.72 | 0.35 | 0.25 | 均衡控场 |
| 虚空针 | 4.8 | 0.58 | 1.35 | 0 | 高穿透 |
| 日冕飞盘 | 3.8 | 0.32 | 0.5 | 0.65 | 旋转多能 |

### 5.2 怪物系统

**5 种基础怪物**:

| 怪物 | HP | 速度 | 伤害 | XP | 出现波次 | 定位 |
|---|---|---|---|---|---|---|
| 碎壳虫 | 18 | 126 | 4 | 2 | Wave 1 | 炮灰 |
| 疾行体 | 24 | 196 | 6 | 3 | Wave 2+ | 快速冲锋 |
| 重甲块 | 88 | 78 | 10 | 8 | Wave 3+ | 坦克 |
| 裂变囊 | 54 | 112 | 7 | 6 | Wave 4+ | 中等威胁 |
| 磁暴卫士 | 160 | 92 | 15 | 13 | Wave 5+ | 精英 |

**怪物缩放** (关键公式):
```
HP 缩放 = 1 + wave×0.035 + combatTime×0.008
伤害缩放 = 1 + wave×0.02 + combatTime×0.0025
生成间隔 = max(0.58, 1.42 - wave×0.045 - elapsed/320)
每批数量 = 5 + floor(wave×0.92) + floor(elapsed/16) + rand(0,3)
怪物上限 = min(620, 210 + wave×10 + cycle×42)
```

### 5.3 经济系统

**双货币**:
- **XP**: 波内临时，用于升级选被动
- **合金 (Alloy)**: 永久货币，用于商店购买 + 机库解锁

**XP 曲线**:
```
升级所需 XP = 500 × 1.34^level + 72 + level×9
```

**商店价格**:
```
物品价格 = max(18, round(15 + tier×8 + floor(wave/5)×2 + (cycle-1)×5))
```

### 5.4 装备系统

**5 个装备槽位**: 头盔 / 胸甲 / 靴子 / 饰品 / 配件

**5 个品质等级**:

| 品质 | 前缀 | 数值倍率 | 成本倍率 | 最高等级 |
|---|---|---|---|---|
| 普通 | (无) | 1.0× | 1.0× | 6 |
| 稀有 | 精制 | 1.42× | 1.72× | 8 |
| 史诗 | 超导 | 2.02× | 2.85× | 10 |
| 传奇 | 星铸 | 2.78× | 4.7× | 12 |
| 神话 | 神话 | 3.66× | 7.4× | 14 |

**每件装备有正面+负面效果** (tradeoff 设计)，确保选择有策略深度。

## 6. 数值平衡基线 (Baseline)

> 参考暴雪方法论: 从核心战斗系统建立锚点，所有其他数值围绕锚点展开。

**锚点假设**: "平均武器 + 平均怪物 = 10 下击杀"

| 指标 | 基线值 | 推导来源 |
|---|---|---|
| 玩家基础攻击力 | ~20 | 武器4.2 × 射速1.8 × 1.05 ≈ 8 |
| 第 1 波怪物平均 HP | ~24 | 碎壳虫 18 × HP缩放 1.33 |
| 第 1 波 TTK (击杀时间) | ~3 秒 | 24 HP / 8 DPS |
| 第 1 波怪物总数 | ~420 | 公式推算 |
| 第 1 波玩家 DPS 覆盖率 | ~22% | 无法全清，合理 |
| 第一次升级时间 | 30-60 秒 | 目标 |
| 第一次购买时间 | Wave 1-2 | 目标 |

**目标波次里程碑**:

| 波次 | 设计意图 | 预期存活率 |
|---|---|---|
| Wave 1-2 | 教学期，轻松，攒资源 | 95%+ |
| Wave 3-4 | 开始有压力，需要走位 | 80% |
| Wave 5-6 | 需要合理 build | 50% |
| Wave 7-8 | 普通玩家极限 | 20% |
| Wave 9-10 | 熟练玩家挑战 Boss | 5% |
| Wave 10+ | 无尽模式，极限挑战 | <1% |

## 7. 数值平衡诊断 (来自真实 Cocos/CDP 跑局)

> 真实跑局脚本: `tools/bot/run_balance_cdp.py`（CDP 驱动真实 Cocos 游戏，不再维护 Python 战斗仿真器）

### 当前问题 (1000 局模拟结果)

| 指标 | 目标 | 当前值 | 判断 |
|---|---|---|---|
| P50 到达波次 | ≥ 6 | 2.7 | ❌ 过低 |
| P90 到达波次 | ≥ 10 | 2.9 | ❌ 过低 |
| 平均升级次数 | ≥ 3 | 0.0 | ❌ 无法升级 |
| 平均购买道具数 | ≥ 4 | 1.2 | ❌ 经济太紧 |

### 根因分析

1. **XP 门槛过高**: 初始 500 XP，每只碎壳虫只掉 2×2.1×0.38 ≈ 1.6 XP → 需要杀 300+ 只才能升级
2. **怪物 HP 时间缩放过快**: `combatTime×0.008` 在 Wave 3 后怪物血量爆炸
3. **经济门槛过高**: 合金掉率低 + 商店价格基线高 = 买不起东西

### 推荐修复 (Balance Patch 1)

| 修改 | 当前 | 推荐 | 理由 |
|---|---|---|---|
| 初始 XP 门槛 | 500 | 80 | 让第一次升级在 30 秒内 |
| XP 曲线倍率 | 1.34 | 1.22 | 避免后期指数爆炸 |
| XP 曲线附加 | 72+level×9 | 28+level×5 | 降低绝对值 |
| HP 时间缩放 | 0.008 | 0.004 | 降低后期血量墙 |
| 伤害时间缩放 | 0.0025 | 0.0012 | 降低后期即死风险 |
| 最小生成间隔 | 0.58 | 0.95 | 给玩家喘息空间 |

## 8. 游戏流程测试框架 (Playtesting)

> 详见: `docs/playtest/PLAYTEST_TEMPLATE.md`

### 四阶段测试法

| 阶段 | 目的 | 方法 | 通过标准 |
|---|---|---|---|
| **1. 概念验证** | 核心循环有趣吗？ | 自己玩 10 局 | 至少 7 局想"再来一局" |
| **2. 数值验证** | 平衡合理吗？ | 真实 Cocos/CDP 跑局 | 分层 P50/P90 达标 |
| **3. 体验验证** | 流程顺畅吗？ | 3-5 人试玩 | 平均爽感 ≥ 3.5/5 |
| **4. 压力验证** | 极端情况 | 150 怪扎堆 / 满屏弹幕 | FPS ≥ 30, 无崩溃 |

### 关键指标 (每局必记录)

| 指标 | 正常范围 | 异常信号 |
|---|---|---|
| 存活时间 | 8-15 分钟 | <5 分钟=太难, >20 分钟=太简单 |
| 首次升级时间 | 30-60 秒 | >120 秒=XP 曲线问题 |
| 首次购买波次 | Wave 1-2 | Wave 4+=经济问题 |
| 死亡原因 | 多样化 | 80%+ 同一原因=系统偏差 |
| 武器选择分布 | 较均匀 | 80%+ 选同一把=其他武器弱 |
| 再来一局意愿 | >70% | <50%=核心体验问题 |
